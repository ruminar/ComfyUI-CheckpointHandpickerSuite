# How to use

## Installation

```bash
cd ComfyUI/custom_nodes
git clone https://github.com/ruminar/ComfyUI-CheckpointHandpickerSuite.git
```

Start or restart ComfyUI.

## The simplest workflow

### Checkpoint cycling mode

Automatically cycle through the checkpoints stored on your SSD and test them in your existing generation workflow.

You do not need to build a new dedicated workflow just to use this suite.

1. Open the image generation workflow you already use.
2. Add a `Checkpoint Name Cycler` node to the canvas.
3. Connect the Cycler's `ckpt_name` output to the `ckpt_name` input of `CheckpointLoaderSimple` or your checkpoint loading node.
4. Connect the Cycler's `ckpt_name_safe` output to the `filename_prefix` input of `SaveImage`.
   - You may combine it with other strings by using string utility nodes, but make sure `ckpt_name_safe` is included in the output filename.
   - This makes it much easier for `ImageDir Preview` to find images generated by each checkpoint later.
5. Set the Cycler's `mode` to `shuffle_once`.
6. Queue as many jobs as you like, such as 100 or 500 jobs.

The Cycler will automatically rotate checkpoints, and the generated images will be saved with checkpoint names included in their filenames.

## A slightly more convenient workflow

### Checkpoint monitoring mode

- Add a `Checkpoint Name Cycler` node as described above.
- Add a `Checkpoint List Selector` node. It does not need to be connected to anything.

The Selector provides the following buttons:

- `🔄Refresh All`

  Use this button after adding or removing checkpoints from your checkpoint directory using Explorer, Finder, or another file manager.

  The checkpoint list can be synchronized without restarting ComfyUI.

- `🏹 Push to Local List`

  This sends the selected checkpoint to the Cycler's Local List.

  When a Cycler has entries in its Local List, normal cycling is temporarily paused and Local List entries are processed first.

## Advanced workflow

### In-job checkpoint review mode

This mode lets you tag checkpoints while a job is running, based on the preview image generated by that checkpoint.

Note:

This feature is intended for users with advanced multi-stage workflows.

For example, it is useful when your workflow generates an intermediate image, then performs additional steps such as hand correction, face correction, upscaling, or post-processing.

If your workflow is a simple one-pass generation workflow, the next job may start immediately after the preview appears. In that case, this mode may not be useful, and you can skip this section.

- Add `Checkpoint Name Cycler` and `Checkpoint List Selector` as described above.
- Add an `Ephemeral Preview` node.
  - Connect it to the `IMAGE` output of a `VAE Decode` node in the middle of your workflow.
- Add a `Checkpoint Status Tagger` node.
  - Connect the Tagger's `ckpt_name_str` input to the Cycler's `ckpt_name_str` output.

Now, while a job is running, you can check the preview image and tag the checkpoint as favorite, nice, keep, or delete-reserved.

Status changes are immediately shared with the Checkpoint Handpicker Suite UI, including the `Checkpoint List Selector`.

`Checkpoint Name Cycler` can also use checkpoint status as a filter when cycling.

### Output-folder checkpoint review mode

This mode lets you keep the main generation workflow running while reviewing generated images in another browser tab.

Note:

In this tab, you do not execute jobs.

This mode only uses the ComfyUI user interface as a review screen. To avoid accidentally queueing jobs from the review tab, it is a good idea to move the queue control panel away from your main working area.

- Keep your main generation workflow running.
- Open ComfyUI in another browser tab.
- Add a `Checkpoint List Selector` node.
- Add a `Checkpoint Status Tagger` node.
  - Connect the Tagger's `ckpt_name_str` input to the Selector's `ckpt_name_str` output.
  - The Selector button will change from `🏹 Push to Local List` to `🎯 Sync Checkpoint`.
- Add an `ImageDir Preview` node.
  - Connect its `ckpt_name_str` input as well.
  - By default, it searches images in the ComfyUI output folder.
  - If you want to use another directory, provide a full path string to `search_directory`.
    - It searches subdirectories and prioritizes newer files.
    - It supports common image formats such as PNG and JPEG.

After that, select a checkpoint in the Selector and press `🎯 Sync Checkpoint`.

The selected checkpoint will be sent to the connected Tagger and Preview nodes, and preview images will be displayed.

When you tag a checkpoint using the Tagger, the status is shared immediately. Other tabs can refresh or update their own Suite UI, and Cycler filters can use the updated status.

## Deleting checkpoints marked for deletion

- A deletion script is generated in the ComfyUI `temp` directory.
- After your overnight batch generation and review work are finished, move to the ComfyUI `temp` directory and run the generated script from a terminal.
  - The script remains available as long as the `temp` directory is not deleted, so you can run it later if needed.

```bash
python delete_reserved_checkpoints.py
```

Checkpoints are not deleted immediately when you press the delete tag.

Deletion is always handled by the generated script, so you can review and confirm the final cleanup manually.

## Checkpoint review as a workflow

Checkpoints are not just files.

Each model has its own strengths, weaknesses, habits, and moments where it works unexpectedly well.

However, checkpoint review often tends to be vague:

- generate a few images
- remember a rough impression
- search for outputs later by filename
- hesitate to delete anything because the decision feels risky

ComfyUI-CheckpointHandpickerSuite was designed to make this process more explicit and repeatable.

The basic ideas are:

1. You need checkpoint cycling before you can efficiently gather review targets.
2. Interesting checkpoints should be easy to revisit by inserting them into the next jobs.
3. Statuses should stay simple: `favorite`, `nice`, `keep`, `delete`, or `none`.
4. `delete` should mean delete reservation, not immediate physical deletion.
5. Review statuses should be usable as filters for future cycling.
6. To make those filters meaningful, checkpoint selection should be resolved at execution time, not when the job is queued.
7. Execution-time resolution makes it possible to follow checkpoint refreshes and status changes without restarting ComfyUI.
8. Review should be possible both during generation and from generated output images in another tab.

In other words, this suite is not only a set of nodes.

It is a practical proposal for discovering, testing, reviewing, revisiting, and safely cleaning up checkpoints in checkpoint-heavy ComfyUI workflows.
