# 🧬 ComfyUI-CheckpointHandpickerSuite (日本語版聖典 v0.2.0)

> ** 「ディスク容量が節約できる保証はありません。なぜなら、あなたが空いた容量に歓喜して、明日さらに多くのCheckpointをダウンロードしてしまうからです。」 **

エクスプローラーとの果てしない往復も、不要モデルを消すためだけの不毛なComfyUIの再起動も、すべては過去の遺物じゃ！
`ComfyUI-CheckpointHandpickerSuite` は、膨大なCheckpoint（.safetensors）の海で戦うAI絵師のために、生成パイプラインを「一瞬も止めず」、常時稼働のままリアルタイムにモデルの格付け・棚卸し・サムネイル設定までを執行するために錬成された、究極のノンストップ・アセット管理システムじゃ。

v0.2.0の特大アップデートで、ついに「失敗画像の直接削除」や「寝転がりながらの外部レビュー（DirectLink）」まで手に入れてしまったぞ！ さあ、狂乱の宴の始まりじゃああ！

* **Checkpointの全自動ノンストップ巡回**
* **Local Listによるピンポイントの割り込みテスト**
* **VRAM直射・キャッシュゼロの超軽量プレビュー（SSD摩耗ゼロ）**
* **👑神（god!）を含む5色のステータスタグによる冷徹な格付け**
* **プレビュー画面からの「失敗画像」ダイレクト物理削除！**
* **神画像を「チェックポイントの代表サムネイル」にワンクリック設定！**
* **タブレットから遠隔でタグを打てる堕落の「DirectLink」モード！**
* **誤爆を防ぐ「delete（削除予約）」キューシステム（サムネイルも一緒に道連れじゃ！）**
* **評価結果に応じた動的な巡回フィルタリング**
* **ComfyUIの再起動を一切要求しない、Refresh Allへの完全追従（実行時遅延解決）**
* **メインの生成を全力疾走させたまま、別タブで優雅に棚卸しを行う非同期マルチタブ運用**

## ❔ これは何なのか？（What is this?）

**「あなたのCheckpoint読み込みノードに機能を追加し、デラックスなレビューデスク付きの豪華なジュークボックスにします。」**

大量のモデルをただの「肥大化したデータファイル」として無機質に扱う時代は終わりました。
このSuiteを導入した瞬間、あなたのComfyUI環境は、世界中の神モデルたちが織りなす「極上のAIアート・ジュークボックス」へと変貌します。

ユーザー（あなた）が行うのは、お気に入りの名盤（Checkpoint）を選ぶことだけ。
寝ている間に自動でレコードが切り替わり（Cycler）、
吐き出された極上のメロディ（画像）を目の前で楽しみながら（Preview）、
「絶対神（👑）」のプレイリストを作るか、その場で「代表サムネイル」に上書き設定するか、不要な失敗画像を消し飛ばすか、あるいはモデルごと「棚卸し（🗑）」にするかを1クリックでジャッジする。

これは、ストレージ難民を救済し、モデルの命の選別をゲーム感覚で行うために生まれた、世界で最も贅沢な「アセット・ソムリエシステム」なのじゃ！

## 🗂 構成要素の説明（The Four Sisters）

我がリポジトリを支える、シャドウガーデンの序列（4人の看板娘たちと5つの魔導具）を紹介しよう。それぞれが固有の役割を持ち、おぬしのComfyUI（肉体）と脳内で非同期にリンクしておるのじゃ！

### 1. 🤎 [Checkpoint Name Cycler（心臓）](CheckpointNameCycler.md)

バックグラウンドで何百件ものジョブをノンストップで爆走させるための動力源じゃ。

* `fixed`（固定）、`increment`（順次）、`randomize`（ランダム）、そして全モデルを1枚のデッキにしてシャッフルし、重複なしで引き切る `shuffle_once` の4モードを完備。
* KSamplerへモデルを渡す直前のフェーズでCheckpoint名を動的に注入する「ジョブの実行時遅延解決（Lazy Resolution）」を搭載。
* さらに、現在のインデックスを汚さずに特定のモデルを最優先で割り込ませる緊急レーン「Local List」をも制御する、本Suiteの文字通りの「心臓」じゃ。

### 2. 💜 [Checkpoint List Selector（頭脳）](CheckpointListSelector.md)

リポジトリ内の全モデルを一手に統括し、状況に応じて役割を3変化させる司令塔へと超絶進化を遂げたぞ！

* 接続先がない時は **`🏹 Push to Local List`** として機能し、Cyclerへ割り込み曲を送り込む。
* 目（Preview）とリンクした瞬間、UI動相同期モードである **`🎯 Sync Checkpoint`** へと動的に覚醒する！
* 右手（Tagger）のみと直結すると、外部レビュー用の **`🌕 DirectLink ON / 🌑 OFF`** ボタンが出現するのじゃ！
* もちろん、新モデルの追加や物理削除の結果を、**ComfyUIの再起動なしで1秒で同期する `🔄 Refresh All` アビリティ**も健在じゃ。マウスホバーでサムネイルがヒュンッと浮かび上がる洒落た機能も追加されたぞ！

### 3. 💛 [Checkpoint Status Tagger（右手）](CheckpointStatusTagger.md)

画像を見ながら、モデルに対して冷徹かつ瞬時に格付け（レイティング）を行う執行官じゃ。

* v0.2.0で絶対神こと `god!(👑)` が降臨！ ボタンは `god(👑)` / `favorite(💛)` / `nice(👍)` / `keep(✔)` / `delete(🗑)` の5つに増強されたぞ！（`none` はトグルで解除）。
* 「👑や💛や👍や✔がついている間は、deleteボタンが物理的にグレーアウトし、サーバー側でも絶対に削除予約を弾く」という、誤爆を100%許さない鉄壁のセーフティバリデーション。右手の誇りにかけて、神モデルを絶対に見捨てぬ設計じゃ！

### 4. 💙 [Ephemeral Preview](EphemeralPreview.md) ＆ [ImageDir Preview（目）](ImageDirPreview.md)

インジェクションされた情報とストレージをマッハで逆探知する、Suiteの「網膜」じゃ。v0.2.0でコイツが一番ヤバい進化を遂げたぞ！

* **Ephemeral Preview:** 中間生成ラインで、今まさに画像を吐き出しているモデルの名前を、フロントエンドのタブローカルな実行状態から検知して美しくラベリング表示する。
* **ImageDir Preview:** 棚卸しモードの主役。Selectorでモデルを選んだ瞬間、過去に出力した画像（サブフォルダ含む）を高速スキャンし、コンタクトシートとしてキャンバスへ一瞬で展開する。
  * **【神機能】** プレビューされた画像を左クリックしてみろ！ なんとアクションメニューが開くのじゃ！
  * **`Delete image`**: おもむろに失敗画像を「物理削除（直接消去）」できる！レビューしながらゴミ掃除が捗るのじゃ！
  * **`Set as checkpoint thumbnail`**: 気に入った画像を、その場で「チェックポイントの代表サムネイル」に上書き設定できる！もうCivitaiからわざわざ画像を落とす必要はないのじゃ！

## 📱 タブレットで寝転がりレビュー！？「DirectLinkモード」

v0.2.0の目玉機能じゃ！「わざわざComfyUIの狭いプレビュー画面でチマチマ見るのは疲れる！」というおぬしのために用意したぞ。

`Checkpoint List Selector` を `ImageDir Preview` に繋がず、直接 `Checkpoint Status Tagger` にだけ繋いでみろ。
するとボタンが **`🌑 DirectLink OFF`** に変わる。
これをON（🌕）にすれば、おぬしはiPadや外部の画像ビューア、共有フォルダでフルスクリーンで生成画像を楽しみながら、PC側のComfyUIでポチポチとリストから選んでタグを打つだけで良くなるのじゃ！
堕落の極みじゃろ？ まさに特等席でのソムリエごっこじゃ！

## 🗑 削除予約の最終執行について

Taggerで `delete（🗑）` を押したモデルは、その場では物理削除されず、安全のためにテンポラリの `checkpoint_delete_queue.jsonl` に予約キューとして蓄積される。
バッチ作業がすべて終わったら、自動生成されている安全執行スクリプト `delete_reserved_checkpoints.py` をコンソールから実行し、対話的に `(y/N)` でパージすればSSDの容量解放は完了じゃ！もちろん、ComfyUIの再起動は不要じゃぞ。

**v0.2.0からは、なんと設定したサイドカーサムネイル画像（.jpgなど）も賢く一緒に消し去ってくれる**ようになったぞ！チリひとつ残さぬ徹底ぶりじゃ！

## 推奨出力ノード

- [`ComfyUI-GMImageSaver`](https://github.com/ruminar/ComfyUI-GMImageSaver)
- [`ComfyUI-PillowImageSaver`](https://github.com/ruminar/ComfyUI-PillowImageSaver)

どちらもCheckpointレビュー向けに同じファイル名規則・ディレクトリ規則で保存するから、環境に合わせて好きな方を選べるようになっておるのじゃ！
おすすめは、もちろん`GraphicsMagick`利用版じゃな！　（注：`GraphicsMagick`の別途インストールが必要です）
外部コマンドなしで使いたい場合は、`ComfyUI-PillowImageSaver`を使うとよいぞ。

これらのノードを使用する場合、`ckpt_name_safe` を、`ImageSaver` の `label` 入力に繋ぐのじゃ。
すると生成された画像はチェックポイントを反映したファイル名やディレクトリで保存され、後でCheckpointの評価の際に参照できるようになるのじゃ！
