# 🔮 陰の思想：Checkpoint評価を、ひとつの「儀式（ワークフロー）」として扱うのじゃ！

おぬしたちが夜な夜なストレージへ引き込む `.safetensors`（Checkpoint）は、単なる容量を食う無機質なデータファイルではない。

それぞれのモデルには、神がかった得意の表現、致命的な苦手構図、独特な愛おしい癖、精度、そして思わぬ場面で奇跡を起こす瞬間が秘められておる。

しかし、そのCheckpointの評価（格付け）はこれまで、どうしても感覚的な闇に葬られがちじゃかった。

何枚か適当に生成して、なんとなく良い・悪いを脳内で覚えておく。

膨大なファイル名を頼りに、後からエクスプローラーで血眼になって探す。

本当に消してよいか迷ったまま、ストレージ（SSD）の肥やしとして残り続ける。

**ComfyUI-CheckpointHandpickerSuite** は、その曖昧になりがちな「棚卸しの試練」を、極めて合理的かつ冷徹なロジックによって、扱いやすい一連の流れ（因果）として整理するために作られたのじゃ！

わらわたちが導き出した、基本となる設計思想（結界の掟）は次の通りじゃ：

1. **巡回なくして選別なし：** まず自動で縦横無尽に巡回できなければ、評価対象（分母）すら効率的に集められんのじゃ。

2. **割り込みのアビリティ：** 巡回中であっても、「おっ」と気になるCheckpointを見つけたら、即座に次のジョブへ最優先で割り込ませられるべきじゃ。

3. **6値の絶対格付け：** 評価ステータスは、人間の脳に負担をかけない `god` / `favorite` / `nice` / `keep` / `delete` / `none` の6値だけでシンプルに完結させるべきじゃ。

4. **慈悲深き処刑宣告：** 不要と判断した `delete` はその場で物理破壊（unlink）するのではなく、神絵の差し戻しを許す「削除予約」であるべきじゃ。

5. **因果のフィルター：** 格付けした評価結果を用いて、次の巡回候補を優雅にフィルタリング（お気に入り狙い撃ち等）できるべきじゃ。

6. **時空の遅延解決（Lazy Resolution）：** そのフィルターを常に有効にするため、Checkpointの選択はジョブ登録時（Queueボタンを押した時）ではなく、**実際に生成ラインが走る「実行直前」に遅延解決（オンデマンド解決）すべき**なのじゃ！

7. **不死身のプロトコル：** 実行時遅延解決であるならば、ComfyUIを再起動させることなくモデルを追加・削除して `Refresh All` をブチ込んでも、システムはエラーを吐かずに最新状態へ追従できるのじゃ！

8. **並列世界の検分：** 評価の執行は、メインジョブの実行中にメイン画面で行うほか、画像作成ジョブを全力疾走させたまま、JavaScriptの無線イベントを用いて、別タブで過去の出力画像をニヤニヤ確認しながら完全非同期で行えるべきじゃ！

9. **多元評価の収束（Unified Review Surface）：** 評価はひとつの画面だけに縛られるべきではないのじゃ。ジョブ実行中にその場で評価してもよし、Checkpoint名から結果画像を参照してじっくり確認してもよし、タブレット端末で生成画像を眺めながらCheckpointを選んで評価してもよし。どの入口から下した審判であっても、最終的には `god` / `favorite` / `nice` / `keep` / `delete` / `none` という同じ評価体系へ収束し、Checkpointと結果画像の対応関係を壊さずに蓄積されるべきなのじゃ！

10. **無侵襲の増設（Non-Invasive Extension）：** これほど強力な非同期評価システムでありながら、既存のメイン生成ラインを1ミリも汚さず、今使っているお気に入りのワークフローの `Load Checkpoint` ノードの頭に `Checkpoint Name Cycler` をただ1個カチッと挟み込むだけで、別タブへの「無線棚卸し環境」が即座に自動展開される。導入のために過去の遺産（既存の配線）を破壊させるような無粋な真似は、我がシャドウガーデンの流儀には反するのじゃよ！


このSuiteは、単にキャンバスをスパゲッティにするための無機質なノードではない。

Checkpointを発見し、試し、評価し、必要なら安全にパージするための、**まったく新しい実用的なワークフローの提案（神アセット）** なのじゃーーーっ！！！🚀🔮✨🌟🏆✨🌐
